# 🕴️ ภารกิจ "ธุรกิจหน้าฉาก" — Capybara Import & Export Co., Ltd.

คุณคือ **Fixer** ของตระกูล ตอนนี้ตระกูลกำลังถูก **สรรพากร** ตรวจสอบ
ทางออกคือต้องมี **"ธุรกิจหน้าฉาก" (front business)** ที่ดูถูกกฎหมาย
ทีมจึงจดทะเบียนบริษัทนำเข้า-ส่งออก **Capybara Import & Export Co., Ltd.**
และต้องรีบทำ **เว็บไซต์บริษัท** ให้เสร็จ

**ภารกิจ:** ลงชื่อ "ทีมผู้บริหาร" ของทุกคนในทีมให้ครบบนหน้าเว็บ
โดยทุกคนแก้ไฟล์ [index.html](index.html) ที่ **บรรทัดเดียวกัน** (บรรทัดที่มี `<!-- TEAM -->`)

> ระหว่างทางคุณจะได้ฝึก GitHub workflow ครบวงจร: **Issue → Branch → Commit → Push → Pull Request → Code Review → Merge** (รวมถึงการแก้ **merge conflict** ของจริง)

---

##  เป้าหมายของงาน (Definition of Done)

งานเสร็จเมื่อ ไฟล์ `index.html` บน branch **`main`**:

- มี **ชื่อทุกคนในทีมครบ** อยู่บน **บรรทัด `<!-- TEAM -->` เดียวกัน** คั่นด้วย comma
- ไม่มี **conflict marker** เหลือ (`<<<<<<<`, `=======`, `>>>>>>>`)
- ไม่มีข้อความ placeholder ในวงเล็บเหลืออยู่

ตรวจด้วยคำสั่ง: `./check.sh 4` (เปลี่ยน `4` เป็นจำนวนคนในทีม)

## 👥 บทบาทในทีม (ทีมละ 4 คน)

| คน | ทำอะไร |
|----|--------|
| **คน 1** | ตั้ง repo ของทีม, เชิญเพื่อน, เปิด Issue, ใส่ชื่อตัวเองลง `main` ก่อน, คุมคิว merge, เป็น reviewer |
| **คน 2** | สร้าง branch → ใส่ชื่อ → เปิด PR → (merge เป็นคิวแรก ไม่ชน) |
| **คน 3** | สร้าง branch → ใส่ชื่อ → เปิด PR → (merge แล้วจะ **ชน** ต้องแก้) |
| **คน 4** | สร้าง branch → ใส่ชื่อ → เปิด PR → (merge แล้วจะ **ชน** ต้องแก้) |

> คำสั่ง git แบบพิมพ์ตามได้ทั้งหมดอยู่ใน **[docs/COMMANDS.md](docs/COMMANDS.md)**

## STEP 1 — ตั้ง repo (คน 1 ทำคนเดียว)

1. กดปุ่ม **"Use this template"** บน GitHub → สร้าง repo ของทีม
2. **Settings → Collaborators** → เชิญ คน 2, 3, 4 เข้า repo
   (คนที่ถูกเชิญต้องกดรับ invite ในอีเมลหรือ GitHub ก่อน)
3. เปิด **Issue** 1 อัน ชื่อ `🎯 ลงชื่อทีมงานให้ครบ` — Issue นี้ใช้สำหรับ:
   - บอก **ลำดับคิว merge** (เช่น คน 2 → คน 3 → คน 4)
   - เช็กว่า PR ไหนได้ **2 approvals** แล้ว
   - **แจ้งเมื่อเกิด conflict**
4. **ใส่ชื่อตัวเองลง `main` ก่อน** (คน 1 ไม่ต้องเปิด PR):
   เปิด `index.html` บน GitHub → กดดินสอ ✏️ → หาบรรทัด `<!-- TEAM -->` → แก้
   ```
   ทีมผู้บริหาร: (คน 1 ใส่ชื่อตัวเองที่นี่ก่อน)
   ```
   ให้เป็น
   ```
   ทีมผู้บริหาร: <ชื่อคน1>
   ```
   → **Commit directly to the `main` branch**

---

## STEP 2 — ใส่ชื่อตัวเอง (คน 2, 3, 4 ทำพร้อมกันได้)

> ทำตามคำสั่งทีละบรรทัดใน [docs/COMMANDS.md](docs/COMMANDS.md)

1. **`clone`** repo ของทีมลงเครื่อง
2. สร้าง **branch ชื่อตัวเอง** เช่น `add-name-michael`
3. เปิด `index.html` → แก้ **เฉพาะบรรทัด `<!-- TEAM -->`** → **ต่อชื่อตัวเองท้ายบรรทัด คั่นด้วย comma**
   - จาก `ทีมผู้บริหาร: Don` → เป็น `ทีมผู้บริหาร: Don, Michael`
   - ❌ ห้ามลบ/ทับชื่อคนอื่น   ❌ ห้ามแก้บรรทัดอื่น
4. **`commit`** การแก้ไข
5. **`push`** branch ของตัวเองขึ้น GitHub
6. เปิด **Pull Request** → ตั้ง **base = `main`**
7.  **ยังไม่ merge** — รอจนทุกคนเปิด PR ครบก่อน

---

## STEP 3 — Code Review (ทุกคน)

ทุก PR ต้องได้ **2 approvals** ก่อน merge

ไป review PR ของเพื่อน แล้วเช็กตาม checklist นี้ ถ้าครบให้กด **Approve**:

- [ ] ใส่ **ชื่อจริงสะกดถูก**
- [ ] แก้ **เฉพาะบรรทัด `<!-- TEAM -->`** (ดูที่แท็บ Files changed)
- [ ] **ไม่ได้ลบชื่อคนอื่น** (ต่อท้ายด้วย comma เท่านั้น)
- [ ] ไฟล์ยังเป็น **HTML ที่เปิดดูได้** (ไม่มี marker ค้าง, tag ไม่พัง)

---

## STEP 4 — Merge ทีละคิว (ตามที่คน 1 ประกาศใน Issue)

merge **ทีละคน** ตามคิว ห้าม merge พร้อมกัน

| คิว | คน | ผลที่เกิด | ต้องทำ |
|-----|----|-----------|--------|
| 1 | คน 2 | **merge ผ่าน (ไม่ชน)** | กด Merge ได้เลย |
| 2 | คน 3 | **ชน (conflict)** | แจ้งใน Issue → แก้ conflict → merge |
| 3 | คน 4 | **ชน (conflict)** | แจ้งใน Issue → แก้ conflict → merge |

คนที่ merge ทีหลังจะชน เพราะแก้ **บรรทัดเดิม** ที่เพิ่งถูกคนก่อนหน้าเปลี่ยนไป

---

## STEP 5 — แก้ conflict (เฉพาะคน 3 และ 4)

เมื่อ GitHub บอกว่า conflict ให้ทำตามชุดคำสั่ง **RESOLVE CONFLICT** ใน [docs/COMMANDS.md](docs/COMMANDS.md)

หัวใจคือ เมื่อเจอแบบนี้ในไฟล์:

```text
<<<<<<< HEAD
<!-- TEAM --> <p id="team">ทีมผู้บริหาร: Don, Sonny</p>
=======
<!-- TEAM --> <p id="team">ทีมผู้บริหาร: Don, Michael</p>
>>>>>>> main
```

**ต้องรวมชื่อทั้งสองฝั่ง** ให้เหลือบรรทัดเดียว แล้วลบ marker ออกให้หมด:

```text
<!-- TEAM --> <p id="team">ทีมผู้บริหาร: Don, Sonny, Michael</p>
```

> ❌ ห้ามเลือกฝั่งใดฝั่งหนึ่งทิ้ง — **ทุกชื่อต้องอยู่ครบ ห้ามมีใครหาย**

---

## STEP 6 — ตรวจงาน

```bash
./check.sh 4
```

(เปลี่ยน `4` เป็นจำนวนคนในทีม) — script จะบอกว่าผ่านหรือยัง และถ้ายังไม่ผ่านจะบอกว่าขาดอะไร

เมื่อชื่อครบทุกคนบน `main` → **ภารกิจสำเร็จ** 🎉
